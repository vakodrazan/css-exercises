Research about Custom properties

  1. We need double dashes (--) characters to define a Custom property's name.
  2. We cannot use spaces and periode (full stop) with the name.
  3. It depends on what value we want to set in with a new Custom property such as number, keywords, text. Any possible value could work.
    • It is useful to create a Custom property :root or html when we want the child to inherit the property's value.
    • When we want a different value in each case then it isn't useful to have Custom property in the :root.
  5. If we've defined a new Custom property we can recall it variable by using a var() function, but we don't need to assign anything anymore.
  6. We can override a Custom property by changing the value, nesting selectors, using @media
  7. Custom properties are so useful because it reduce repitation, write less code, one point of control it can help us saving our time when we edit the same things.
  8. Custom properties only work in a declaration block.
