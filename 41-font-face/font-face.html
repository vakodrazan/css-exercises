Font face

  1. There are several types of font formats, these are some of them:
    • Woff(Web Open Font format)
    • Woff2(Web Open Font format2)
    • TTF (True Type)
    • Otf(Open Type)
    • Eot(Embedered Open Type)
    • Svg(Scalable Vector Graphics)
  2. What I make of it is woff and woff2 are good candidate to use in all browser.
  3. Difference between a @font-face, we have to use src: url() or local() and @keyframes, it needs an animation name and have special character like from / 0% and to / 100% / @media queries / @supports blocks
  4. Some fonts come with licencing restriction means they are free for everyone to use but it need to download.
  5. If we want to support all the chosen fonts in here, we need to to provide eight font-face.
  6. At roughly 20kb per font file, our users would have to download 160KB in the example above.
  7. If we chose to offer WOOF2 as well as WOOF, we need to use the the scr: ; it has two url and two format('the name'). In each file we need to use the both font, according to the question 5 we only need eight font-face.
  8. Most of our users would probably have the exact font we want to serve already installed on their devices. We can use the local(the font's name) to make sure that the user's browser doesn't download the fonts they might already have.
  9. Sometimes we have to assign fallback fonts for our custom webfonts we add via @font-face.
  10. The browser can use one of several strategies for loading the fonts, e.g. showing nothing until the font is downloaded, or using a fallback font first. We can use:
    • font-family
    • font-size
    • font-weight
    • font-style
